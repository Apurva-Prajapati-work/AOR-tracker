## 🤔 Why AORTrack Exists

IRCC's official processing time tool gives you a **single, generic number** — the same "6 months" for every CEC applicant, regardless of when they applied or how their cohort is performing.

**That's not good enough when your whole life is on hold.**

AORTrack replaces that vague estimate with **crowd-sourced, cohort-specific data** from real applicants who submitted in the same stream and same month as you. You'll know:

- Where you rank in your cohort (e.g. *Top 28% of CEC Jan 2024*)
- What day your cohort started receiving PPR requests
- Whether processing times are speeding up or slowing down *right now*
- Your realistic earliest, median, and latest PPR date

All free. All open source. No account needed.

## ✨ Features

### 📊 Live Community Feed
> *Real-time updates from 14,847+ applicants — not a weekly IRCC report.*

Stream of live PPR receipts, milestone updates (BIL, BGC, medicals), and processing events. Filter by stream, cohort month, or milestone type. Every entry is submitted by a real applicant and cross-referenced with cohort data.

---

### 🗂 Personal Dashboard
> *Your command centre — days elapsed, cohort rank, PPR probability.*

The moment you enter your AOR date, you get a personalised dashboard showing:
- **Days since AOR** benchmarked against your cohort's median
- **Cohort rank** — where you sit as a percentile (e.g. Top 28%)
- **Live cohort breakdown** — PPR'd vs medicals vs still waiting
- **Average PPR day** for your stream and month, updated daily

> *Stop guessing. Know your window — earliest, median, and latest.*

Built from real cohort velocity data, not IRCC's static estimates. Your PPR window updates automatically as new community data arrives.

## 🚀 Quick Start

### Use the Live App *(no setup)*
Visit **[track.getnorthpath.com](https://track.getnorthpath.com)** and click **Track My AOR**. Done.

### Run Locally

**Prerequisites:** Node.js 18+, npm or yarn

```bash
# 1. Clone the repo
git clone https://github.com/Get-North-Path/AOR-tracker.git
cd AOR-tracker

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
# Edit .env.local with your config (see Environment Variables below)

# 4. Start the development server
npm run dev

# App is now running at http://localhost:3000
```

### With Docker

```bash
# Build and run
docker compose up --build

# App at http://localhost:3000
```

## ⚙️ Environment Variables

Create a `.env.local` file from the template:

```bash
cp .env.example .env.local
```

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | ✅ | PostgreSQL connection string |
| `NEXT_PUBLIC_APP_URL` | ✅ | Your app URL (e.g. `http://localhost:3000`) |
| `SMTP_HOST` | ☑️ | SMTP server for email alerts |
| `SMTP_PORT` | ☑️ | SMTP port (usually `587`) |
| `SMTP_USER` | ☑️ | SMTP username |
| `SMTP_PASS` | ☑️ | SMTP password |
| `DISCORD_WEBHOOK_URL` | ☑️ | Discord webhook for alert notifications |
| `DISCORD_BOT_TOKEN` | ☑️ | Discord bot token for community features |
| `CRON_SECRET` | ☑️ | Secret for securing cron job endpoints |
| `NEXT_PUBLIC_POSTHOG_KEY` | ❌ | PostHog analytics (optional) |

> ☑️ = Required for specific features. The core tracker works without email/Discord config.

## 🗄 Database Schema *(simplified)*

```prisma
model AOREntry {
  id          String   @id @default(cuid())
  stream      Stream   // CEC | FSW | FST | PNP | AIP
  aorDate     DateTime
  itaDate     DateTime?
  cohortKey   String   // e.g. "CEC-2024-01"
  
  // Milestones (null = not yet received)
  bilDate     DateTime?
  bgcDate     DateTime?
  medicalsDate DateTime?
  pprDate     DateTime?

  // Meta
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([cohortKey])
  @@index([stream, aorDate])
}

model CohortSnapshot {
  id          String   @id @default(cuid())
  cohortKey   String   @unique
  stream      Stream
  cohortMonth DateTime
  totalCount  Int
  pprCount    Int
  medCount    Int
  bgcCount    Int
  medianPprDay Int?
  fastestPprDay Int?
  snapshotAt  DateTime @default(now())
}
```

## 🤝 Contributing

### Ways to contribute

- 🐛 **Report a bug** — [Open an issue](https://github.com/Get-North-Path/AOR-tracker/issues/new?template=bug_report.md)

- 📊 **Add your AOR data** — [Track your application](https://track.getnorthpath.com/track) and contribute to the dataset
- 💬 **Help others** — Join [Discord](https://discord.gg/aortrack) and answer questions
- ⭐ **Star this repo** — It helps more applicants find us

### Development Workflow

```bash
# 1. Fork the repo, then clone your fork
git clone https://github.com/YOUR_USERNAME/AOR-tracker.git

# 2. Create a feature branch
git checkout -b feat/cohort-histogram-tooltip

# 3. Make your changes & write tests
npm run test

# 4. Lint & type-check
npm run lint
npm run type-check

# 5. Commit with a conventional commit message
git commit -m "feat(cohort): add tooltip to processing day histogram"

# 6. Push and open a PR against main
git push origin feat/cohort-histogram-tooltip
```

### Commit Convention

We use [Conventional Commits](https://www.conventionalcommits.org/):

| Prefix | When to use |
|--------|-------------|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `data:` | Dataset or calculation change |
| `docs:` | Documentation only |
| `style:` | Formatting, no logic change |
| `refactor:` | Code restructure |
| `test:` | Adding or fixing tests |
| `chore:` | Build, CI, tooling |

### Good First Issues

New to the codebase? Look for issues labelled [`good first issue`](https://github.com/Get-North-Path/AOR-tracker/issues?q=label%3A%22good+first+issue%22) — these are scoped, well-documented, and mentored.
